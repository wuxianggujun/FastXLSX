# FastXLSX OPC Editing

Public facade 是 `WorkbookEditor` / `WorksheetEditor`；package reader/editor、EditPlan、dependency/relationship graph 是 internal。

- Unchanged/unknown part copy-original。
- Changed part stream/small-part rewrite 或 remove。
- 功能必须声明 preserve/audit/fail/edit 与 relationship/content-type/calc side effects。
- 跨 edit plan、replacements、omitted entries 和 manifest 的 metadata mutation 必须先 staging，再以 noexcept commit 发布；测试需注入提交前失败并验证原状态与 retry。
- Existing-workbook `add_worksheet()` 只创建 generated empty worksheet，并同步 workbook catalog、workbook relationships、content types、manifest 和 generated part；source catalog 保持不变。它不等于 clone，且新增表保存重开前只能走 Patch 填充/rename，不能直接 In-memory materialize。
- Existing-workbook `remove_worksheet()` 只删除 relationship-closed worksheet；首版拒绝 last-visible、active/selected、definedNames、formula、materialized、queued payload、owned relationship 和 extra-inbound 风险，并同步 workbook catalog、workbook relationship、content type、manifest 与 worksheet entry。Unsupported linked semantics 默认 fail，不静默 orphan 或 repair。
- Existing-workbook `add_internal_hyperlink()` 只重写 target worksheet XML 中的 worksheet-local `<hyperlink location>`；它保留 existing worksheet relationships/unknown entries，不创建 `.rels`、content type 或 external relationship。两次 bounded scan 分别规划 schema boundary/duplicate overlap 和写出 staged file；PackageEditor 完成 caller source consumption 后外层临时文件由 RAII 回收，最终 owned replacement file 仍由事务状态管理。
- Existing-workbook `add_external_hyperlink()` 同时重写 target worksheet XML 与 source-owned worksheet `.rels`：分配 owner-local relationship id、写出 `TargetMode="External"`，缺失 `.rels`/`xmlns:r` 时生成必要结构，并保留既有关系/unknown entries。worksheet part replacement、relationship entry replacement、manifest/edit-plan 与 public diagnostics 必须在副本中 staging 后统一 commit；目标 reachability、relationship repair/pruning 与 linked-object semantic sync 仍不属于该窄切片。
- Existing-workbook `add_data_validation()` 只重写 target worksheet XML 中的 `<dataValidations>`：Streaming/Patch 共用 owning rule validator/serializer，Patch 支持 multi-range `sqref`、formula1/formula2、prompt/error escaping、新建/追加/自闭合 container，并严格校验已有 count 与直接子节点一致。它不创建 `.rels`/content type、不请求重算或同步 row/column/cell structural mutation；worksheet replacement 与 public rule count 必须 staged 后统一 commit。
- Existing-workbook `set_auto_filter()` / `clear_auto_filter()` 只整体重写 target worksheet XML 的 root `<autoFilter>`：两次 bounded scan 定位完整 existing element 或 schema-safe rank-5 boundary，set 不保留旧 criteria/sort children，clear absent 不 stage replacement。Table-local filter part、worksheet `.rels`、content types、calc metadata 与 unknown entries 保留；worksheet replacement 与 changed/optional-range public diagnostic 必须 staged 后统一 commit。
- `save_as()` 不是 in-place；pending staged state 与 unsaved watermark 分离。无 options overload 保留 stored 输出；显式 `WorkbookEditorSaveOptions` 可选择 active backend/DEFLATE，配置失败必须早于 dirty-session staging。
- Copy-original 始终要求 logical payload/CRC preservation；production minizip-ng 在 source/output method 匹配时还会复制 exact compressed payload。性能证据必须分列 source/output compressed bytes 与 planned raw names/count/bytes，并验证 payload equality；不得描述成 local header、central directory、extra fields 或整包 byte passthrough。
- Production minizip-ng 对具备完整 expected size/CRC32 的 staged file chunks 在正常路径合并 CRC 并校验 completed-entry CRC；combined mismatch 才重读定位具体 chunk，不完整 metadata 保留 per-chunk fallback。PackageEditor single-pass transform 在 append output 时同步累计 CRC，并在 root/dimension 边界生成 exact file-range checkpoints，commit 不再为 CRC 重读 temporary worksheet；dimension memory chunk 单独携带 CRC。PackageWriter completed-entry CRC、same-size mutation diagnosis、output protection 与 retry 不得删除。PackageEditor 在 minizip-ng profile 通过 `mz_crypt_crc32_update` 计算 staged/fused CRC，stored-only profile 保留 internal portable fallback；`FASTXLSX_ENABLE_PORTABLE_CRC_PROFILING` 只允许相同 minizip package backend 下的 private A/B，不授权直接链接/export `ZLIB::ZLIB`。
- Windows production minizip-ng 对至少 4 MiB staged file chunk 使用两个固定 512 KiB buffer 做 bounded overlapped read-ahead；先发起下一块读取，再同步写当前块。小 chunk、raw compressed-copy、stored-only 与非 Windows 路径保持同步读取；本地 RAII 必须取消并等待 pending IO，不得留下后台 worker。性能报告需分列 activation、prefetched chunks/bytes、1 MiB peak buffer、foreground read/wait、file IO buffer、writer input peak/call count/maximum call wall time，并且 warmed run 不能冒充物理 OS cache-hit 证据。
- Internal direct-zlib raw engine 只有 `FASTXLSX_ENABLE_DIRECT_ZLIB_PROFILING=ON` 才编译；production/stored/no-images 默认 OFF、不直接链接/export ZLIB。启用后只在 minizip backend、非 stored/raw-copy、完整 staged CRC metadata 且 entry 至少 4 MiB 时生效；其有界 output buffer 为 16 KiB–4 MiB 的 2 次幂，其他路径回退 minizip。默认/public backend 不变。Engine CPU 只覆盖 `deflateInit2`/`deflate`/`deflateEnd`；CRC、read、raw-output write 与 entry close 必须分列，backend 采用决定需要 matched-level balanced validated evidence。
- DEFLATE strict existing-cell replace 在无 worksheet relationships、目标全部存在且具有 top-level dimension 时使用 one-inflate + owned temporary-file ranges；missing-cell upsert、relationship-bearing worksheet 与其他 fallback 使用 256 KiB output-batched single-pass source-order transform，在同一扫描中完成精确 dimension、relationship audit 与 telemetry。Internal opt-in Patch reader 会在 generic parser 前识别 bounded window 内 exact writer-compatible value、formula、cached-formula 与 simple inline-string complete cell，并暴露一个 callback-lifetime exact-byte span；attribute 变体、rich/unsupported metadata、malformed markup 与跨窗口 candidate 保留结构 parser 和 diagnostics。Canonical `<is><t>`/`xml:space="preserve"` payload 仍使用不复制的 literal terminator/suffix matcher。Transformer 可继续合并连续 row/raw/untouched-cell span；metadata、replacement、非连续 offset 与窗口边界必须 flush。Batch summary 必须保留 coordinate extrema 与 formula/sharedStrings/style/inline-string audit。有序 upsert 使用单游标，乱序 source 保留 set fallback。性能报告需拆 parser/source-callback/coalesced/action traffic、aggregate/canonical inline-string、complete-cell/canonical complete-cell count/bytes/type counters、pass-through batch count/cells/bytes/peak cells、append/flush/peak buffer、relationship/temporary IO 和 package writer target entry；多 worksheet 顺序 rewrite 还需聚合每个 rewritten worksheet entry 的 input/read/write/close wall/process CPU，并验证 sheet names/value/dimension、按表 edits/inserted、CRC segments 与 superseded temporary file 回收。
- 多 worksheet scaling 使用独立 schema-v2 balanced runner：`fixed-shape` 固定每表 rows/cells/edits，`fixed-total` 固定总 cells/edits并明确 row-number/XML-size 差异；1/2/4-sheet variants 必须 position 等频，保留同轮 baseline ratio、逐 worksheet entry 跨 run 分布与 OpenPyXL，不以 noisy dirty-revision 结果授权并发或 backend 变更。Benchmark schema-v20 的 entry 数组必须按 package 顺序，并与 rewritten aggregate/sheet1 legacy 总账一致；stored backend 不伪造 minizip entry telemetry。
- Dirty In-memory projection 可在写包前进入 internal Patch stage，但只有 package write 成功后才能清 dirty、增加 handoff count 和推进 watermark；失败 retry 必须用当前 CellStore 覆盖 stale stage。
- In-memory 默认拒绝已知有损 materialization，并以 public typed diagnostic 报告稳定语义分类和 source cell context；显式 lossy 才拍平。
- Preservation 不等于 tables/drawings/comments/VBA/pivot/custom XML semantic editing。
